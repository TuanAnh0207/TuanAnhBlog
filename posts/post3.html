Java Collections – chọn DS nào?

So sánh ArrayList, LinkedList, HashMap, TreeMap, HashSet, TreeSet với tình huống thực tế: CRUD, tìm kiếm, sắp xếp, duyệt.

ArrayList vs LinkedList

ArrayList tối ưu truy cập theo index; LinkedList tốt cho chèn/xoá giữa danh sách.

HashMap vs TreeMap

HashMap nhanh trung bình; TreeMap có thứ tự tăng dần theo key (log n).

HashSet vs TreeSet

HashSet kiểm tra tồn tại O(1) trung bình; TreeSet có thứ tự và hỗ trợ range query.

// Ví dụ chọn cấu trúc dữ liệu
List names = new ArrayList<>(); // chèn cuối và truy cập index nhanh
Map scores = new HashMap<>(); // tra cứu theo key nhanh
Set sorted = new TreeSet<>(); // cần dữ liệu đã sắp xếp

Độ phức tạp truy cập

ArrayListget O(1), add cuối O(1)*, remove middle O(n)
LinkedListget O(n), add/remove đầu O(1)
HashMapput/get trung bình O(1)
TreeMapput/get O(log n), có thứ tự

Mẹo thực tế

  • Ước lượng kích thước ban đầu: new HashMap<>(expected*1.33f) để giảm rehash.
  • Dùng Collections.unmodifiableList khi muốn bất biến.
  • Dùng EnumMap/EnumSet nếu key là enum.

FAQ

ConcurrentHashMap vs HashMap? ConcurrentHashMap an toàn luồng; HashMap nhanh hơn trong đơn luồng.

Case study: Feed mạng xã hội

Queue yêu cầu (LinkedList/ArrayDeque), map user→posts (HashMap), giữ thứ tự theo thời gian (TreeMap hoặc PriorityQueue) để phân phối feed.
